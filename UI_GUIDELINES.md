# UI Guidelines

## Navegacao

- Sidebar escura com contraste alto.
- Item ativo sempre visivel.
- Mobile usa drawer.
- Desktop permite modo recolhido.

## Formularios

- Formularios longos devem ser agrupados por secao.
- Edicoes secundarias devem preferir drawer.
- Acoes destrutivas precisam manter cor de perigo.
- Inputs usam foco visivel e labels permanentes.

## Operacao

- Clientes da fila devem aparecer como cards.
- Acoes frequentes precisam de botoes com icones.
- Loading deve aparecer dentro do botao acionado.
- Cards devem preservar informacoes essenciais: nome, senha, posicao, horario e status.

## Tabelas

- Usar tabela apenas quando comparacao tabular for realmente melhor.
- Preferir cards/listas para rotinas operacionais.
- Linhas devem ter hover e hierarquia de texto clara.

## Display publico

- Tipografia grande.
- Alta legibilidade a distancia.
- Modo TV reduz ruido.
- Overlay de chamada deve ter prioridade visual total por 5 segundos.
